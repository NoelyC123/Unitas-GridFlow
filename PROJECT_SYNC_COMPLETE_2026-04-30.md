## KEY INSIGHTS DOCUMENTED

### 1. The Additive Value Proposition (CORRECTED)

**Why this correction matters:**

If GridFlow tries to replace Field Maps' basic record display, it sets up a competition GridFlow will lose (Field Maps is mature, enterprise-grade GIS).

If GridFlow adds interpretation ON TOP OF the same record display, it creates unique value that Field Maps doesn't provide.

**The 5 layers GridFlow must provide:**
1. What records are on the map (parity with Field Maps)
2. What this means for design (interpretation)
3. What is missing (completeness)
4. What is provisional (confidence)
5. What must be checked (QA gates)

### 2. The Height vs. Specification Problem

**Critical discovery:**
> "A proposed pole cannot always have a physically measured height because it does not exist yet."

**Solution:**
- Existing poles: Measured height missing = BLOCKER
- Proposed poles: Specification missing = REVIEW REQUIRED
- Context records: Height field hidden

### 3. Shape ≠ Color Paradigm

**Visual design principle:**
- Shape = Asset Type (what it is)
- Color = Review Status (what state it's in)
- Must be visually decoupled

### 4. Proximity Is Not Proof

**Safe wording:**
> "GridFlow should use proximity as a prompt for human review, not as a final engineering conclusion."
